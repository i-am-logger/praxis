# HTTP -- HTTP Method Semantics Ontology

Models the HTTP request methods as an entity set with the semantic properties — safety, idempotence, body-permission — that RFC 9110 attaches to each method. Rich types `Request`, `Response`, `StatusCode`, `StatusClass`, and `Connection` sit above the ontology; the `engine` drives a connection state machine over them.

Key references:
- RFC 9110 (2022) *HTTP Semantics* (supersedes RFC 7230-7235)
- RFC 9112 (2022) *HTTP/1.1*
- Fielding 2000: *Architectural Styles and the Design of Network-based Software Architectures* (REST)

| Category | Entities |

## Category

Defined via `define_ontology!` as `HttpMethodCategory { concepts: Method, relation: MethodPair }`. Method pairs form the morphism set, generated by `define_ontology!`; qualities (`IsSafe`, `IsIdempotent`) project the RFC 9110 §9.2 semantic properties.

## Qualities

| Quality | Type | Description |
|---|---|---|
| IsSafe | () | Safe methods — no side effects: Get, Head, Options (RFC 9110 §9.2.1) |
| IsIdempotent | () | Idempotent methods — same result if repeated: Get, Put, Delete, Head, Options (RFC 9110 §9.2.2) |

## Axioms (1)

| Axiom | Description | Source |
|---|---|---|
| SafeImpliesIdempotent | Every safe method is idempotent | RFC 9110 §9.2.2 |
