# Quantity -- SI dimensions (abelian group), units, dimensional analysis

Models physical quantities as a value paired with a dimension, where dimensions form an abelian group under multiplication generated by the seven SI base dimensions. Units sit on top of dimensions as a scale factor + dimension pair, and conversion between compatible units is a roundtrip-invertible operation. The axioms verify the abelian-group laws on dimensions, the dimensional-consistency rule for addition, the derived-dimension calculus for products and quotients (e.g. velocity = length / time), and the unit-conversion roundtrip and incompatibility checks.

Key references:
- BIPM 2019: *SI Brochure* (9th ed.) — the 7 base dimensions and unit definitions
- QUDT: *Quantities, Units, Dimensions and Types* ontology (alignment target)

## Entities (8 base dimensions)

| Category | Entities |
|---|---|
| Mechanical (3) | Length (L), Mass (M), Time (T) |
| Electromagnetic / thermodynamic (2) | ElectricCurrent (I), Temperature (Θ) |
| Chemical / photometric (2) | AmountOfSubstance (N), LuminousIntensity (J) |
| Identity (1) | Dimensionless (1) |

## Category

Discrete category over the eight `BaseDimension` entities. The full algebraic structure (the abelian group generated by these) lives in the `Dimension` type, where each value is a tuple of integer exponents over the seven SI base dimensions; the axioms verify the group laws on canonical dimensions (DIMENSIONLESS, LENGTH, MASS, TIME, VELOCITY, ACCELERATION, FORCE, ENERGY, FREQUENCY).

## Axioms (8)

| Axiom | Description | Source |
|---|---|---|
| DimensionCommutativity | [A]·[B] = [B]·[A] | abelian group |
| DimensionAssociativity | ([A]·[B])·[C] = [A]·([B]·[C]) | abelian group |
| DimensionIdentity | [A]·1 = [A] | abelian group |
| DimensionInverse | [A]·[A]^{-1} = dimensionless | abelian group |
| AdditionRequiresSameDimension | meters + seconds = error | quantity calculus |
| DerivedDimensionConsistency | velocity = length / time: [v] = L·T^{-1} | quantity calculus |
| UnitConversionRoundtrip | km -> m -> km is invertible | BIPM SI Brochure |
| IncompatibleUnitConversionFails | METER.convert(.., &SECOND) returns None | BIPM SI Brochure |

Plus the auto-generated structural axioms from `define_ontology!` (category laws on the discrete dimension category).

## Files

- `ontology.rs` -- Entity (BaseDimension), discrete category, DimensionSymbol quality, 8 axioms, tests
- `dimension.rs` -- `Dimension` type: integer exponent vector over the 7 base dimensions, abelian-group operations (multiply, inverse, is_dimensionless), constants (LENGTH, MASS, TIME, VELOCITY, ACCELERATION, FORCE, ENERGY, FREQUENCY, DIMENSIONLESS)
- `unit.rs` -- `Unit { dimension, scale }`: `to_si`, `from_si`, `convert`; named units (METER, KILOMETER, SECOND, …)
- `value.rs` -- `Quantity { value, dimension }`: dimension-checked add and div
- `system.rs` -- Measurement system classification (SI base, derived, etc.) per BIPM SI Brochure
- `constants.rs` -- Physical constants as ontological `Quantity` values
- `tests.rs` -- additional tests beyond `ontology.rs`
- `mod.rs` -- module declarations
